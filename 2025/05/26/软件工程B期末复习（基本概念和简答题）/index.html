软件工程B期末复习（基本概念和简答题）

一.基本概念题

1.什么是软件

软件是能够完成预定功能和性能，并对相应数据进行加工的程序和描述程序及其操作的文档。
或：是计算机系统中与硬件相互依存的另一部分，包括程序、数据、文档、服务的完整集合。

2.什么是软件危机

软件危机是指在软件开发、维护过程中所存在的一系列严重、或致命的问题。目前，几乎所有软件都不同程度地存在这类问题。

3.什么是软件工程

软件工程是开发、运行、维护和修复软件的系统方式，是一门工程学科，即采用工程的概念、原理，技术和方法来开发和维护软件。

4.什么是软件生命周期

软件生命周期又称作软件生存周期、系统开发生命周期，是指从提出开发软件产品开始，直到软件报废为止的全过程，主要包括软件定义、开发和运行三个阶段。

5.什么是模块化

模块化就是把程序划分成独立命名且可单独访问的模块，每个模块完成一个子功能，即相对独立，又相互联系，把这些模块集成起来构成一个整体，可以完成软件系统所要求的功能。模块化是解决软件复杂问题的一种手段。对于一个复杂的大型软件系统，应该将他适当分解，即把复杂的问题分解成许多容易解决的小问题，原来的问题也就解决了。

6.什么是详细设计

软件详细设计的目标是对目标系统做出精确的设计描述。

7.程序内部文档主要包括哪三部分

程序内部的文档包括恰当的标识符、适当的注释和程序的视觉组织等等。

8.什么是软件测试

软件测试是为了寻找错误而运行程序的过程。软件测试应该是对软件开发过程中形成的文档、数据以及程序进行的测试，而不仅仅是对程序进行的测试。

9.什么是黑盒测试

黑盒测试是已知产品的功能设计规格，可以进行测试证明每个实现了的功能是否符合要求。只关心软件的功能是否正确。

10.什么是白盒测试

白盒测试也叫玻璃盒测试。是对软件的过程性细节做细致的检查。这一方法是把测试对象看作一个打开的盒子，它允许测试人员利用程序内部的逻辑，结构以及有关信息，来设计或选择测试用例，对程序所有逻辑路径及逆行测试。

11.什么是测试用例

测试用例是为了高效率地发现软件缺陷而精心设计并执行地少量测试数据。
从测试用例本身构成的角度来看，测试用例是测试执行的最小实体，是为特定的目的而设计的一组测试输入、执行条件和预期结果。

12.什么是软件调试

软件调试是在测试发现错误之后排除错误的过程。

13.什么是软件维护

软件维护是软件生命周期最后一个阶段，其基本任务是保证软件长期正常运行。

14.什么是面向对象

面向对象是一种具体的软件开发技术与策略，是一整套关于如何看待软件系统与现实世界的关系，用什么观点来发现问题并进行求解，如何进行软件系统构造的软件方法学。

15.什么是统一过程

统一过程是一个软件开发过程，是一个将用户需求转化为软件系统所需的活动的集合。

16.什么是软件构架

软件构架是一个系统的组织结构，包括：系统分解的各个组成部分、各部分相互之间的连接性，交互机制、以及指导系统设计的相关规则。

17.什么是设计模式

设计模式是对软件设计中普遍出现的一类问题的解决方案，这种解决方案明确、文档充分、经历时间考验。

18.什么是类

类是对象的模板，是一组具有相同属性和相同操作的所有对象的集合。

19.什么是集成测试

集成测试也称组装测试或联合测试。是指在单元测试的基础上，将所有模块按照要求组装成一个完整的系统进行的测试。组装模块的方式有两种：渐增式测试和非渐增式测试。

20.什么是软件重用

重用又叫复用，是指同一事务不做修改或稍加修改就可以多次重复使用。

21.什么是多态性

多态性是面向对象技术最根本的特征，表明类在统一外表下表现出的多种行为的能力。

22.CASE的英文全称，中文含义

CASE：Computer Aided Software Engineering，中文含义：计算机辅助软件工程

23.什么是回归测试

回归测试是指重新执行已经做过测试的某个程序，以保证所做的程序变化没有带来非预期的副作用。

24.什么是需求分析

软件系统的需求分为用户需求和系统需求两类。软件需求阶段的任务就是要将客户等提出的用户需求转换为系统需求。

25.UML的中文含义和英文全称

UML：Unified Modeling Language 中文含义：统一建模语言
UML主要由UML语义和UML表示法两部分构成

26.单一职责原则

一个类只完成一个功能，或只有一个职务。

27.面向对象技术的四要素

抽象、分解、分类、复用。

二.简答题

1.如何消除软件危机

  1. 理解软件的概念：软件是程序、数据及相关文档的完整集合。
  2. 应该推广使用在实践中总结出来的开发软件的成功的技术和方法。
  3. 应该开发和使用更好的软件工具。
  4. 软件工程学的一个重要目标就是提高软件的可维护性，减少软件维护的代价。

2.软件工程师应该具备的4项技能

抽象：业务、功能的抽象。
分类：归纳、演绎。
分解：复杂系统的分解。
复用：提高分析、设计、开发效率。

3.软件开发有哪三项基本策略

软件复用，分而治之，软件优化与折中。

4.软件生命周期包括哪些阶段

软件周期主要包括软件定义、开发和运行三个阶段。

5.常用软件开发过程模型有哪5个

常见的软件工程模型有：瀑布模型、快速原型模型、演化模型、螺旋模型、喷泉模型等。

6.软件可行性分析的目的和任务分别是什么

7.需求分析的基本任务是什么

软件需求阶段的任务就是要将用户等提出的用户需求转化为系统需求。

8.系统设计包括那两个阶段

结构设计、过程设计。

9.总体设计的主要任务是什么

总体设计的主要任务是：设计软件的结构——也就是确定系统中每个程序是由哪些模块组成的，以及这些模块相互间的关系。

10.衡量模块独立性的两个标准是什么

衡量模块独立性一般采用耦合和内聚两个定性的技术指标进行衡量。

11.模块独立性追求的设计目标

12.人机界面设计主要有哪3项黄金规划

  1. 置用户操作于控制之下
  2. 减少用户的记忆负担
  3. 保持界面一致

13.详细设计工具也称为程序算法设计工具，它主要分为哪3类

表达过程规格说明的工具叫做详细设计工具，也称为算法设计工具。它可以分为三类：图形工具、表格工具、语言工具。

14.软件测试的目的与基本任务是什么

  • 测试目的：设计一系列的测试方案，发现程序中的错误，是为了证明程序有错，而不是证明程序无错。
  • 测试任务：是在规定的条件下对程序进行操作，以发现程序错误，衡量软件质量，评估是否能满足软件设计要求的过程。

15.软件调试的目的是什么

软件测试的目的是尽可能多地发现程序中的缺陷，但这不是最终目的，再进行了成功的测试之后必须进行调试工作。而调试的目的是确定错误的原因和位置，并进行程序修改、排除错误，因此调试也称为纠错。

16.软件维护一般分为哪几类

按软件维护的性质不同，可以把软件维护分为正性维护、适应性维护、完善性维护、预防性维护四种类型。

17.什么是软件维护的副作用

18.简述文档的重要性

文档是影响软件可维护性的决定性因素。所以维护期间的文档比程序代码更重要。

19.软件工程的目的是什么

提高软件的可维护性，减少软件维护所需要的工作量，降低软件系统的总成本。

21.业务模型主要体现为哪两类模型

业务模型是理解一个组织中业务过程的技术。业务模型对应用领域的业务过程进行描述，如：银行的业务过程包括存款、贷款以及投资等。业务模型主要通过业务用例模型和业务对象模型来体现，业务模型提供对客户业务的整体了解。

22.为什么需要构架

需要构架的原因主要有以下四个：理解系统、组织开发、鼓励重用、演化系统。

23.如何描述构架

构架描述是系统各种模型的视图，即是用例、分析、设计、实施和实现模型的视图。构架描述叙述了系统的组成部分，有利于所有开发人员和其他项目相关人员理解系统。

24.什么是用例图？用例与用例之间的关系有哪三种

用例图是由参与者，用例以及它们之间的关系构成的用于描述系统功能的图。
用例与用例之间的关系：扩展、包含、泛化。

25.采用面向数据流建模时，对DFD的精化要精化到什么程度才停止精化工作？

对DFD的精化可以精化到每个加工处理过程只执行一个单一功能为止。

26.什么是结构化程序设计？

是指只用三种基本控制结构（顺序、选择和循环）就能实现任何单入口、出口的程序。另外，，为了方便实际编程，还经常使用 DO-UNTIL 和 DO-CASE 两种控制结构。

27.月收入<=800 元者免税，现用输入数 800 元和 801 元测试程序，请问这是采用黑盒测试法中的哪种方法？请解释其理由？

==是边界值分析方法==。题目中输入的 800 元是采用的刚好等于边界的数据，而 801 选择的是大于边界的数据，因此采用的测试方法是黑盒测试法中的边界值分析方法。

28.现有一个计算类型的程序，它的输入只有一个 Y，其范围是-50≤Y≤50。现从输入的角度考虑设计了一组测试用例：-100，100，0。请问设计这组测试用例的方法是黑盒测试法中的什么方法？并解释其理由。

==是等价类划分方法==。在给定的一组测试用例-100、100、0 中，数据-100 因为小于 Y 的最小值-50，故是无效数据；数据 100 因为大于 Y 的最大值，故也是无效数据；数据 0 应为介于-50 到 50 之间，故数据 0 则是有效数据。因此该方法是黑盒测试法中的等价类划分方法。

29.软件测试时应该追求的目标是什么？软件测试有哪 5 种测试策略？

以最少的测试数据、时间和人力，最大可能的发现更多的错误点。
单元测试、集成测试、确认测试、系统测试、验收测试。

32.软件维护一般分为哪几类？

按软件维护的性质不同，可以把软件维护分为正性维护、适应性维护、完善性维护、预防性维护四种类型。

33.在进行包的设计中，有一条原则是“避免循环依赖”，如果设计时出现循环依赖，会有什么现象发生？

若出现循环依赖则使得任何一个包都不能独立的重用，修改任何一个包都会引起所有包的变化。

35.用例实现指的是什么

用例实现即用交互图来实现参与某用例的对象的交互和协作建模——即每个用例的每个事件流都需要建立一个顺序图，每个用例都需要建立一个与之对应的类图。

36.衡量软件结构好坏的指标有哪几种？

深度、宽度、扇出数、扇入数。

37.E-R 图中的“联系”主要有哪几种方式？请分别针对每一种方式进行举例说明。

（1）一对一联系：如：班主任和班级【注：作弊按 0 分处理】
（2）一对多联系：如：教师与课程【注：作弊按 0 分处理】
（3）多对多联系：如：学生与课程【注：作弊按 0 分处理】

38.系统流程图与程序流程图有什么区别

系统流程图表达的是信息在系统中各个部件之间流动的情况。
程序流程图表达的是对信息进行加工处理的控制过程。

39.什么是设计元素？设计元素主要包含哪5个？

设计元素是指能够直接用于实现的模型要素。
包、设计类、子系统、接口、主动类。

40.报的设计原则是什么？

避免循环依赖、避免分支依赖。

41.如何确定边界类？如何确定控制类？

边界类：在用例模型中，每一对”参与者/用例“之间可定义一个边界类。
控制类：在用例模型中，通常，每个用例至少定义一个控制类，业务复杂的可能要分解为多个控制类。

42.调试和测试有什么区别？

测试是为了发现错误，而调试是为了找到错误位置。
测试以已知条件开始，使用预先定义的程序，有预知的结果，而调试以不可知内部条件开始，结果一般不可预见。

43.使用基本路径测试法进行软件测试时主要包括哪三个关键步骤？

  1. 根据程序代码或流程图，画出相应的程序图
  2. 计算流图的环形复杂度V（G）=路径数目？
  3. 确定只包含独立路径的基本路径集合。
