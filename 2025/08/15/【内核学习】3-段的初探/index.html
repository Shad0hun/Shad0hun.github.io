WinDbg指令概述

我们首先来介绍一下WinDbg中的一些指令：

  • r系列
    r指令用来查看寄存器内容，直接显示所有寄存器的值
    r eax可以用来查看eax寄存器的值
    r gdtr显示GDTR的内容（Global Description Table，全局描述符表），当我们执行该命令的时候，WinDbg会调用调试引擎读取当前CPU上的GDTR的内容并显示

  • d系列
    d系列命令用来显示内存内容，d就是dump的意思
    d后面是可以带单位的，如：db以字节形式查看，dw以字形式查看，dd以双字形式查看，dq以qword形式查看
    d后面还可以带描述，如：dp以指针形式查看，du以无符号字符串形式查看，dc以ANSI C字符串形式查看，ds尝试将内存指针解释为符号，dW以Unicode形式查看
    当然，单位和描述还可以一起用

  • e系列
    e系列命令用于把指定的值写入到目标内存，它是调试时修改内存/打补丁/写入字符的常用工具
    e系列之令可以带单位，如：eb以字节形式写入，ew以字形式写入，ed以双字写入，eq以qword形式写入
    e系列之令也可以带描述：如：ep以指针形式写入，等同于ed、eq，ea、eza以ASCII字符串形式写入（带NULL），eu、ezu以Unicode形式写入（带NULL），ef写入单精度浮点型
    另外e命令会继承上一次写入的格式

三.段选择子

段选择子（Segment Selector） 是 x86/x86-64架构中段式内存管理的关键字段之一，它本质上是一个16位的值，用于在段寄存器（如 CS, DS, SS, ES, FS, GS）中标识段描述符在GDT/LDT中的位置，以及访问权限。换句话说：段选择子是 CPU 用来“选出”某个段描述符的地址和权限的索引号。一个段选择子包含一下项目：

  • Index (3~15位)
    指明GDT或LDT中的描述符编号（从0开始计数）。实际偏移量 = Index × 8（因为每个描述符 8 字节）
  • TI（第2位）
    表示是从 GDT 还是 LDT 中取描述符：0 → GDT，1 → LDT
  • RPL（前2位）
    请求特权级（0 ~ 3），用于特权级检查。结合 CPL（当前特权级）和 DPL（描述符特权级）决定是否允许访问。这个我们后面还会详细讲到。
    那我们来通过实践来看一下某个段选择子，我们就选择我们调试的时候最常见的ds:0x23吧：
    首先，我们对其进行拆分：
    0000 0000 0010 0011
    首先拆分出RPL：11
    然后是TI：0，说明我们要去GDT表中去寻找该段描述符
    最后是Index：0000 0000 0010 0，这个是4，也就是说在GDT表中，第4项就是我们要找的段描述符。

四.段描述符

段描述符是放在GDT/LDT里面的表项，用来描述一个“段”的基址(base)、界限(Limit)、类型与权限等。
段寄存器中只有一个段选择子，CPU会用它去GDT/LDT中找到对应的段描述符，然后把描述符的内容加载到“隐藏部分”（Description Cache）参与寻址/权限检查。
我们来看一下段描述符的结构：

更直观地分块（按比特）：

  • Limit 15:0（bits 0–15）
  • Base 15:0（bits 16–31）:确定该段的第0字节在4GB线性地址空间中的位置。
  • Base 23:16（bits 32–39）
  • Access Byte（bits 40–47）：指明段或者门的类型，确定段的范围权限和增长方向。
    • bit7 P（Present）：指明段是否存在
    • bit6..5 DPL（Descriptor Privilege Level, 0~3）：指明该段的特权级。
    • bit4 S（Descriptor Type，1=代码/数据段，0=系统段/门）
    • bit3..0 Type（低 4 位），这个type还有一张对应表指明该段到底是什么段
  • Limit 19:16（bits 48–51）
  • Flags（bits 52..55
    • bit55 G（Granularity）：0=字节粒度，1=4KiB 粒度
    • bit54 D/B：数据/栈段默认操作数或栈宽度（16/32）；对代码段是默认操作数大小（16/32）
    • bit53 0
    • bit52 AVL：供软件使用
  • Base 31:24（bits 56–63）

Base 计算：Base = Base[31:24] << 24 | Base[23:16] << 16 | Base[15:0]
Limit 计算：Limit = Limit[19:16] << 16 | Limit[15:0]

因为我们修改了段描述符，而段描述符中有一个Base，这里ds:[4E700]指向的地址实际上时ds段的Base+0x4E700，而我们将Base字段修改之后，这里实际指向的地址不是原来的value，所以导致取值错误。
那为什么最后打印的时候又是对的呢？因为我们在打印之前，已经将ds段选择子复原了，所以最终打印是正确的，当然，如果Base偏移稍微多一点都会出错。
