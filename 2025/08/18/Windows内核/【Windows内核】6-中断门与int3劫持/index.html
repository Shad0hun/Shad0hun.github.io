一.中断

中断(Interrupt)是计算机系统中一种硬件/软件基址，用来打断CPU当前正在执行的任务，把控制权转移到一个特定的中断服务程序(Interrupt Service Routine,ISR)，执行完中断服务后再返回原程序。
它的存在意义在于：让CPU不必“死等”外设或者错误事件，而是等事情发生时在被“提醒”。这大大提高了计算机的并发能力和效率。
在Intel x86模式下，中断会通过IDT(Interrupt Description Table)找到对应的中断门/陷阱门/任务门，再由CPU自动完成栈切换、寄存器压栈、标志位控制等操作。
从可否屏蔽的角度，中断可以划分为两位：

  • 可屏蔽中断
    来源：主要来自于外部硬件设备（例如键盘、网卡、磁盘控制器、定时器等）
    线信号：在x86上通过INTR引脚进入CPU。
    特点：CPU是否响应，取决于EFLAGS.IF位
  • 不可屏蔽中断
    来源：主要是严重的硬件错误或系统级事件。
    线信号：zaix86上通过NMI引脚进入CPU
    特点：NMI不能被EFLAGS.IF控制，CPU必须响应

另外，需要注意的是：我们调试的时候进行的断点，这不是中断，而是异常，这两个点特别容易混淆。

二.中断发生，CPU对EFLAGS的处理

当 CPU 响应一个中断（可屏蔽、不可屏蔽、异常都算），它会自动做两件关键事

  1. 把当前的 EFLAGS 压栈保存 ——这是为了保证中断返回时能恢复原先的执行状态。
  2. 修改 EFLAGS 中的某些位，其中最重要的是：
    • 如果是通过 中断门 (Interrupt Gate) 进入：CPU 会自动清除 IF (Interrupt Flag) 位，屏蔽后续的可屏蔽中断。
    • 如果是通过 陷阱门 (Trap Gate) 进入：CPU 不会清 IF，允许继续响应新的中断
      这样设计的目的是：
  • 中断门 → 保证 ISR 不会被别的可屏蔽中断打断，避免混乱。
  • 陷阱门 → 适合调试器、异常处理（允许嵌套）。

当 ISR 执行 IRET（或 IRETD/IRETQ）指令返回时：

  • CPU 会把之前压栈的 EFLAGS 弹出，恢复到中断发生前的状态。
  • 这意味着：
    • 如果中断发生前 IF=1，那么返回时也会恢复成 1。
    • 如果中断发生前 IF=0，那么返回时也会是 0。

三.中断门概述

在Intel x86保护模式下，中断门(Interrupt Gate)是一种系统段描述符，用于把CPU得控制权从当前执行的程序转移到中断或异常处理程序。他的核心目的就是在发生中断或者异常的时候，CPU自动地切换到相应的处理流程，以保证系统能够正确地应对外部设备请求、异常错误或软件触发的中断。
和调用门一样，中断门也是定义在中断描述符表中的一类“门描述符”，但是他们的应用场景不同。中断门最大的特征是：当CPU通过中断门进入处理程序时，会自动清除EFLAG寄存器中的IF位，从而屏蔽进一步可屏蔽中断。这保证了当前中断例程不会再被新的外部中断打断，避免嵌套混乱。

四.中断门描述符


这里我们可以看到和调用门是比较像的，我们来看一下其中标志位的含义：
Offset：目标中断处理程序入口偏移
Selector：目标代码段段选择子
Type：这里和调用门一样，都是固定的，根据系统描述符表可知在中断门中，32位下，该值为1110也就是14.
DPL：描述符特权级，一般设置为0
P：描述该段是否有效，必须为1

五.IDT表

中断描述符表(IDT)是x86架构下的一张数组表，每一个表项对应一个中断向量号。当CPU接收到中断/异常/NMI/软中断的时候，就会用中断向量号作为索引，去IDT中找到对应的描述符，然后跳转到指定的中断/异常处理例程。
IDT是CPU硬件层面强制使用的基址，软件必须经过正确初始化IDT，操作系统才能正常工作。Linux，Windows内核启动时，都会构建自己的IDT，并通过LIDT命令加载到CPU的IDTR寄存器中。
IDT本质上是一张门描述符表，其中包含的类型主要有三类：

  • 中断门
  • 陷阱门
  • 任务门
    当CPU遇到中断/异常的时候，会生成一个中断向量号，然后作为索引差IDT
  • 0–31 (0x00–0x1F) → Intel 保留，用于异常和系统故障（如 0 除、缺页异常、通用保护错误）。
  • 32–255 (0x20–0xFF) → 系统可用，用于硬件中断和软件中断
    比如：
  • 向量 0 → 除零错误 (#DE)
  • 向量 14 → 缺页异常 (#PF)
  • 向量 32 (0x20) → 时钟中断（通常 OS 映射给 PIT 或 APIC Timer）
  • 向量 128 (0x80) → Linux 经典系统调用入口 (int 0x80)

八.陷阱门

陷阱门是x86保护模式下的一种门描述符，存放在IDT中，它的作用和中断门一样：把CPU从当前执行流程转移到一个特权级更高的中断/异常处理例程。
而他俩最大的区别在于：

  • 中断门会清除EFLAGS.IF位，禁止后续可屏蔽中断进入
  • 陷阱门不会清空IF位，也就是说在进入处理程序后，仍然允许新的中断或者异常嵌套触发。

    陷阱门描述符格式与中断门几乎相同，主要区别就是Type位不同。
