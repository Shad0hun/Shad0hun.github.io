【Windows内核】1.双机调试环境搭建

在学习内核的时候，由于我们需要去观察操作系统内核中的结构体等，我们需要一个双机调试的环境，这里就来记录一下双机调试环境的搭建。这里主要是Windows 7的环境搭建，首先我们需要搭建好虚拟机，下载好WinDbg软件。

工具了解

首先，我们需要了解一些工具：

1.bcdedit

bcdedit是Windows操作系统中的一个命令行工具，用于查看和修改启动配置数据（BCD）。启动配置数据存储重要的启动信息，包括启动加载程序和启动设置。这个工具主要由高级用户、系统管理员和开发人员使用，以调整与系统启动相关的各种参数。
工具界面：

我们在bcdedit中的命令，实际上都是写到winload的配置里面去的，而不是注册表，所以我们需要用到这个工具去配置环境。

2.msconfig

MSConfig是用于在Windows操作系统中配置软件，设备驱动程序，引导参数和启动服务的工具。 它与Windows XP和以前的操作系统的Microsoft System Configuration Utility一起发布。
工具界面：

观察这个界面，我们发现该界面是没有办法去创建引导的，所以我们需要结合上面介绍的bcdedit工具去创建一个引导。

被调试端配置

  1. 首先，我们将当前引导状态复制一份，命名为debug：
    bcdedit /copy {current} /d debug
    这样的话我们将当前引导状态复制了一份，并命名为debug，需要注意的是，在这条命令成功执行之后，会生成一个id，这个id我们需要复制，并且保证后面命令中的id与改id一致
  2. 将我们生成的启动项加入到启动菜单中，并放到最后：
    bcdedit /displayorder {14b2d2bb-2107-11eb-b8b2-ae737953d857} /addlast
  3. 设置全局调试通道为串口，端口为com1，波特率为11520：
    bcdedit /dbgsettings SERIAL DEBUGPORT:1 BAUDRATE:115200
  4. 对该启动项开启内核调试，内核加载后调试器可接管：
    bcdedit /bootdebug {7db2d2bb-2107-11eb-b8b2-ae737953d857} ON
  5. 对该启动项开启阶段调试，也就是在bootmgr和winload这些早期组件阶段久允许调试器连接：
    bcdedit /debug {7db2d2bb-2107-11eb-b8b2-ae737953d857} ON
  6. 设置启动菜单的停留时间为30秒，让我们有时间去选择调试模式：
    bcdedit /timeout 30
    设置完这些后，被调试端就设置好了，现在我们需要去调试端，也就是我们的物理机器上去设置。

调试端设置

首先，我们需要移除虚拟机中的打印机，以免调试端口被占用，然后添加串行端口：

最后，我们需要去设置windbg，让其可以调试我们设置好的虚拟机：
设置Windbg的目标：
"C:\Program Files (x86)\Windows Kits\10\Debuggers\x86\windbg.exe" -y "SRV*D:\symbols(win7)*http://msdl.microsoft.com/download/symbols" -b -k com:port=\\.\pipe\com_1,baud=115200,pipe
“C:\Program Files (x86)\Windows Kits\10\Debuggers\x64\windbg.exe”是本快捷方式关联的windbg可执行文件的绝对路径。
“\\.\pipe\com_1”是虚拟机添加串行端口时使用的命名管道名称，这个要与之前Windows7虚拟机的设置一致。
“D:\symbols(win7)”是我们使用的符号文件默认保存位置，可以根据自己喜好进行设置。
