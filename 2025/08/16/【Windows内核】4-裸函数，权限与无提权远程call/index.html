==裸函数实际应用：==
编写钩子（Hook）代码
- 裸函数常用于内联汇编跳板，实现对 API 的劫持或内核函数挂钩。
- 方便直接控制寄存器和调用栈，不受编译器插入的额外指令影响。
实现中断/异常处理程序
- 在内核编程中，ISR（中断服务例程）或 IDT（中断描述符表）指向的处理函数需要精确控制入口/出口，裸函数很适合。
实现特定 ABI 或跨平台接口
- 某些底层接口需要特定的栈布局，裸函数允许开发者完全定制。
性能优化
- 在极少数情况下，为了避免编译器生成多余的栈操作，可以用裸函数手写高效汇编。

二.RPL、DPL与CPL详解以及在访问控制中的作用

1.CPL（当前特权级）

CPL 是处理器当前执行代码的特权级，范围 0~3。
数值越小，权限越高：
0 → 内核态（最高权限）
3 → 用户态（最低权限）
那么我们如何获取CPL的值呢？实际上，CPL 存储在 CS（代码段寄存器）的段选择子低 2 位中（实际上ss段的段选择子也可以，但是这两者肯定是相同的）。
在任意时刻，CPU 的 CPL 是唯一的，代表 CPU 当前执行环境的安全等级。

2.RPL（请求特权级）

我们前面的章节在介绍段选择子的时候，遇到过RPL这个字段。
RPL 是 段选择子（Segment Selector）低 2 位的字段，范围 0~3。
它表示 调用者在访问该段时声明的特权级，也可以理解为“，表示访问者希望以何种特权级访问该段。
RPL 不代表 CPU 当前的运行特权级（那是 CPL 的作用），而是访问者在访问特定段时附带的权限标记。

3.DPL位（描述符特权级）

DPL 是段描述符中用于描述该段所需的最低特权级的2位字段，取值范围为 0~3，数值越小，权限越高，表示访问该段所需的最低特权级别。
在IA-32架构的保护模式下，DPL是实现分段访问控制的关键要素之一，与CPL（Current Privilege Level） 和 RPL（Requested Privilege Level） 共同参与段访问权限的判定。
表示该段的特权级别（Privilege Level）：
0 → 最高权限（内核态）
3 → 最低权限（用户态）

特权级别访问控制中的作用

1. 数据段访问规则

访问数据段时，处理器会计算有效特权级：
Effective Privilege Level = max⁡(CPL,RPL)
数据段访问条件为：
Effective Privilege Level ≤ DPL
若不满足该条件，则处理器触发 GP（General Protection Fault） 异常。
另外如果是堆栈段，需要DPL = RPL = CPL

2. 代码段访问规则

代码段分为 一致（Conforming） 和 非一致（Non-Conforming） 两类，DPL 判定规则不同：

  • 非一致代码段：
    CPL = DPLL
    调用者的 CPL 必须与段的 DPL 相等，才能进行控制转移。
  • 一致代码段：
    CPL ≥ DPL
    低权限代码可以调用高权限的一致代码段，但执行过程中CPL不会改变（无权限提升）。

特权级别的设计意义

  1. 隔离用户态与内核态资源
    典型做法是将内核段 DPL 设置为 0，用户段 DPL 设置为 3，从而防止用户态直接访问内核段。
  2. 控制特权接口调用
    通过设置代码段 DPL，可以定义哪些调用者有权执行特定段内的例程。
  3. 支持多级安全模型
    在需要多层次安全防护的系统中，可以使用 0~3 四个等级分别为不同模块分配访问权限。
