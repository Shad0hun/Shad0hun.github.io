Fuzzing101-4：libTIFF（代码覆盖率）

在本节练习中，我们将尝试对LibTIFF图像库进行模糊测试，目标是在libtiff 4.0.4中找到CVE-2016-9297的崩溃POC，并测试崩溃的代码覆盖率数据。

1.预期收获

通过本节练习，我们的预期收获是：掌握如何使用LCOV测量代码覆盖率并掌握如何使用代码覆盖率数据来提高模糊测试的有效性。

2.下载并构建目标程序：

这个步骤在前面见过很多遍了，这里只给出操作命令：

安装完成后测试：
./tiffinfo -D -j -s -w '/home/shad0hun/桌面/Fuzzing101/04/tiff-4.0.4/test/images/logluv-3c-16b.tiff'

3.下载安装lcov

代码覆盖率是一种软件指标，显示的是每一行代码被触发的次数。通过使用代码覆盖率，我们将了解模糊测试器到达了那些代码，并可视化模糊测试的过程，在使用之前，我们需要安装lcov：
sudo apt install lcov

4.使用lcov重新构建libtiff

现在我们需要使用--coverage标志重建libTIFF：

现在我们使用lcov来收集一下代码覆盖数据：

我们先来解释一下每一条指令：
lcov --zerocounters --directory ./：重置之前的计数器
lcov --capture --initial --directory ./ --output-file app.info：返回包含每条检测线的零覆盖率的“基线”覆盖率数据文件，这个文件是这样的：

$HOME/fuzzing_tiff/install/bin/tiffinfo -D -j -c -r -s -w $HOME/fuzzing_tiff/tiff-4.0.4/test/images/palette-1c-1b.tiff：运行要分析的应用程序（可以多次输入运行）
lcov --no-checksum --directory ./ --capture --output-file app2.info：将当前覆盖状态保存到app2.info文件中
最后，我们需要生成HTML输出：
genhtml --highlight --legend -output-directory ./html-coverage/ ./app2.info
如果一切顺利，代码覆盖率报告已经生成于./html-coverage文件夹下面，只需要打开./html-coverage/libtiff/index.html，就可以看到代码覆盖率内容：

在这里我们不仅可以浏览不同的文件夹和文件，还可以查看被执行的次数。

7.代码覆盖率分析

我们现在来挑一些样本来看看代码覆盖率：
重置之前的计数器：locv --zerocounters --directory ./
返回包含每条检测线的零覆盖率的”基线“覆盖率数据文件：lcov --capture --initial --directory ./ --output-file app.info
运行程序：/home/shad0hun/桌面/Fuzzing101/04/tiff-4.0.4-Fuzzing/bin/tiffinfo -D -j -c -r -s -w '/home/shad0hun/桌面/Fuzzing101/04/out/default/crashes/id:000000,sig:06,src:000000,time:16258,execs:6662,op:arith16,pos:55,val:-2'
将当前覆盖状态保存到id0.info中：lcov --no-checksum --directory ./ --capture --output-file id0.info
生成HTML文件：genhtml --highlight --legend -output-directory ./html-coverage/ ./id0.info
==这里要注意一个问题，就是我们在fuzz结束之后，我们得使用lcov构建的应用程序跑样本，这样才能得出代码覆盖率，否则会报错缺少一个.cov文件，我在这里也十分花费了好长时间，也可以同时保存使用locv和ASan构建出来的版本，ASan版本用来做Fuzz，lcov构建出来的版本用来测试代码覆盖率==
我们先来用一个样本来看看代码覆盖率：

我们从这个可是画图中就可以看到各个文件的覆盖率，也可以点进文件，看看到底那些代码被执行了。
现在我们将所有样本都跑一边，看看总的代码覆盖率：

我们来看一下最后的代码覆盖率，看起来好像其实并不是很高，这可能是因为生成的应用程序有很多，我们只Fuzz了一个应用程序，也有可能是afl的问题，我们需要改造afl，但是这都不是我们现在要考虑的问题，当我们学会改造afl的时候，就可以通过改造afl来提高代码覆盖率。
