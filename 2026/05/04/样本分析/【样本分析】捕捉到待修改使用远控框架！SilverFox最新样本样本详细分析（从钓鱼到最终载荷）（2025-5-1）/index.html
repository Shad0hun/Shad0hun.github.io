概述

本样本整体呈现出典型的多阶段投递 + 白加黑执行 + ShellCode分层加载 + 反射式PE注入的复杂攻击链结构，从初始伪装安装包到最终远控模块加载，形成完整的隐蔽执行体系。
该样本采用了典型的“安装器伪装 + 分阶段解密执行”的架构：

  • 第一阶段：NSIS Installer 伪装载体
    • 使用 NSIS（Nullsoft Scriptable Install System）封装
    • 通过脚本驱动自动解压与执行
    • 内嵌 stub payload 作为解包入口
  • 第二阶段：NSIS Payload Loader
    • 解析 installer stub（NullsoftInst 标记）
    • 动态定位 payload 起始偏移
    • 进行内存申请 + 解压 + 校验执行流
    • 进入虚拟机解释器（NSIS VM）
  • 第三阶段：白加黑执行链
    • 利用合法签名程序（GameBox / sysdiag）
    • 通过 DLL 劫持加载 Utility.dll
    • 触发导出函数执行恶意逻辑
  • 第四阶段：System级持久化与提权
    • Token 复制（OpenProcessToken + DuplicateTokenEx）
    • Session ID 绑定（WTSGetActiveConsoleSessionId）
    • CreateProcessAsUser 提权执行
    • 注册 Service（httpsvc）
  • 第五阶段：ShellCode Loader
    • AES-256-CBC 解密 payload（BCrypt API）
    • VirtualAlloc + VirtualProtect 执行
    • CreateThread 方式启动
  • 第六阶段：网络拉取第二阶段Payload
    • Winsock TCP 通信
    • 分块接收 + XOR 解密
    • 动态拼装 ShellCode
  • 第七阶段：反射式 PE Loader
    • 手动解析 PE Header
    • 修复重定位表（Relocation Fix）
    • 重建 Import Table
    • 直接调用 EntryPoint 执行
  • 最终阶段：远控框架模块
    • 模块化 DLL（load / run / shangxianbiaoji）
    • 具备上线回传 + 指令执行能力
    • 典型 C2 远控架构

背景

在逛论坛的时候，发现了一个样本：Silver Fox#火绒安全 白加黑(出处: 卡饭)，由于正好五一放假闲来无事，来详细分析一下（剖析清楚恶意软件的功能，我感觉非常爽，哈哈哈哈哈哈哈啊哈），看一看SilverFox最近有没有用什么新技术。
