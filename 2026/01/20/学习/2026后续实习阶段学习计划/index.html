周计划

周次 学习方向 学习内容细化（技术视角） 实践安排（工程/源码） 阶段目标与产出
第1周 密码学与认证机制 TLS/SSL 协议流程、证书链验证逻辑、常见认证机制的安全假设与实现边界 结合协议流程与真实漏洞案例，分析实现层面导致安全问题的原因 能从实现角度理解认证与加密相关漏洞成因
第2周 Fuzz 总体原理 覆盖率反馈模型、插桩机制（编译期/运行期）、Fuzz 的适用边界） 网上查阅相关资料从阅读AFL相关源码进行 建立 Fuzz 技术体系整体认知
第3周 AFL 工程实践 AFL 执行流程、样本队列管理、forkserver 机制 对文件解析程序进行 AFL Fuzz，观察覆盖率变化 掌握 AFL 基本使用与调试方法
第4周 AFL 源码理解 覆盖率 bitmap 设计、变异策略调度、queue/favored 逻辑 阅读 afl-fuzz.c 核心代码，复现并分析 crash 从源码层面理解 AFL 工作机制
第5周 AFL工程使用 参数调优、持久化、ASAN/QEMU 对真实解析程序进行 AFL Fuzz 能稳定、高效地“用好 AFL”
第6周 libFuzzer 实战 In-process Fuzz 模型、Sanitizer 协同机制 编写 libFuzzer harness，对目标函数进行 Fuzz 能为指定代码模块构建 Fuzz 测试
第7周 libFuzzer 原理（源码） LLVM 插桩、Coverage tracing、Mutation engine 调度逻辑 阅读 libFuzzer 关键源码模块，结合实际 Fuzz 行为进行验证 理解 libFuzzer 核心实现原理，提升 harness 设计质量
第8周 协议 Fuzz 原理 协议解析流程、字段校验顺序、状态机推进逻辑 结合协议或解析模块源码，定位解析入口与关键状态迁移路径 掌握协议类目标的 Fuzz 基本方法
第9周 协议 Fuzz 优化 字段依赖处理、无效输入剪枝、覆盖率瓶颈分析 根据源码逻辑优化输入构造，提高有效覆盖率 提升协议 Fuzz 的有效性
第10周 网络程序 Fuzz 网络服务执行模型、本地化 Fuzz 思路 本地部署网络服务，构建可 Fuzz 的测试环境 掌握网络程序 Fuzz 的基本流程
第11周 状态机问题 会话状态维护、状态机复杂性 结合源码分析状态依赖，调整 harness 或测试方式 能在有状态场景下稳定运行 Fuzz
第12周 网络漏洞分析 网络 crash 触发条件与调试方法 复现并定位一次网络异常或漏洞 形成完整的网络 Fuzz 分析能力
第13周 高级 Fuzz 理论 污点分析、Hybrid Fuzz 的设计思路（理解级） 阅读相关实现或论文，结合已有实践理解其适用场景 拓展对 Fuzz 技术边界的认知
第14周 Fuzz 工程能力 执行效率优化、参数调优、AFL++ 扩展机制 对已有 Fuzz 项目进行性能优化 提升 Fuzz 工程化能力
第15周 技术边界认知 覆盖率指标局限性、不适合 Fuzz 的目标类型 总结低收益或失败的 Fuzz 场景 建立对 Fuzz 技术的理性判断
第16周 理解现代检测体系 & 构建测试环境 - 杀软/EDR 检测维度（静态、动态、行为、云）
- MITRE ATT&CK 中 Defense Evasion 技术映射
- 搭建多杀软+EDR测试环境（Win10/11 VM）		 1. 安装 Defender + 1–2款商业杀软（如 Kaspersky、Bitdefender）
2. 部署 Elastic Security 或 Velociraptor 模拟 EDR
3. 编写简单 Meterpreter 载荷，观察检测点		 - VirusTotal 扫描结果
- 本地杀软是否告警
- EDR 是否记录进程行为
第17周 掌握 Shellcode 加载与基础静态免杀 - Shellcode 提取与加密（XOR/AES）
- 自定义 PE Loader（C/C++）
- 字符串/导入表混淆
- 时间戳/节区名修改		 1. 用 MSFVenom 生成 shellcode
2. 编写 C loader：VirtualAlloc + memcpy + call
3. 对 shellcode 加密，运行时解密
4. 修改 PE 头字段规避 YARA 规则		 - VT 检出率下降
- PE 文件通过 pestudio 无高危特征
第18周 绕过 AMSI 与 ETW（Windows 特有机制） - AMSI 工作原理与内存 Patch
- ETW 日志机制与禁用方法
- 直接调用 Nt* API 替代 Kernel32		 1. 编写 C/C++ 程序：Patch AmsiScanBuffer
2. 禁用 ETW 事件（通过 EtwEventWrite patch）
3. 使用 SysWhispers3 生成 syscall stub		 - PowerShell 脚本可执行未被拦截
- Process Monitor 显示无 AMSI 调用
第19周 进程注入与动态行为免杀 - APC / Hollowing / Early Bird 注入
- 间接系统调用（HellsGate / Halo’s Gate）
- 控制流混淆（避免典型 API 序列）		 1. 实现 Process Hollowing 注入 calc.exe
2. 改用 NtCreateThreadEx + syscall
3. 添加 Sleep(10) + 用户交互检测（GetCursorPos）		 - EDR 未触发“可疑远程线程”告警
- Procmon/API Monitor 无 CreateRemoteThread
第20周 无文件攻击与 .NET 免杀 - PowerShell / MsBuild / Regsvr3 利用
- .NET Assembly 反射加载（Assembly.Load）
- ConfuserEx / Obfuscar 混淆		 1. 用 Donut 生成 .NET payload
2. 编写 C# 反射加载器
3. 混淆后通过 msbuild.exe 执行		 - 无磁盘落地 exe
- Defender 不拦截 .csproj 执行
第21周 反沙箱与反分析技术 - 虚拟机检测（CPUID、注册表、MAC）
- 延迟执行 + 环境感知
- 动态解密关键逻辑		 1. 编写检测 VirtualBox/VMware 的函数
2. 若检测到沙箱，退出或休眠
3. 关键 shellcode 在运行时从远程 HTTPS 解密		 - ANY.RUN 中载荷不执行 payload
- 本地正常触发反向连接
第22周 绕过商业 EDR（实战模拟） - 分析 EDR Hook 机制（Inline Hook / IAT Hook）
- 直接 syscall + unhooking 技术
- 内存权限隐藏（PAGE_NOACCESS）		 1. 使用 SharpSploit 或自研工具 unhook ntdll
2. 用 direct syscalls 执行敏感操作
3. 尝试 Cobalt Strike BOF 思路实现轻量载荷		 - 在 Elastic Security / CrowdStrike Falcon 模拟环境中不触发告警
第23周 综合项目：构建全链路免杀载荷 - 整合前7周技术：
• 加密 + 反沙箱 + AMSI/ETW bypass + syscall注入
- 输出可复现的 PoC		 开发一个 C++ 载荷：
1. 启动时检测沙箱/调试器
2. Patch AMSI/ETW
3. 从资源段解密 shellcode
4. 通过 syscall 注入 notepad.exe		 - 本地多杀软+EDR 环境静默执行

第一周（2026年1月19日）

了解认证类型：

认证类型
密码认证 OWASP 认证基础
Token 认证（JWT/API Key） JWT.io 简介
证书认证（mTLS/ZTNA） Cloudflare 客户端证书说明
多因素认证（MFA） Yubico MFA 介绍
另外，本周需要对毕业设计做一个规划，思考有哪些模块，创新点等等
了解编译期插桩机制（阅读源码afl-gcc、afl-as.c、afl-as.h)
学习AFL覆盖率反馈模型（可以逆向插桩后的test.c，查看其插入的代码）
